Hi,
We've been having issues in the graphical installer for quite a while,
and I only found time to dig into it during DebConf. I've managed to
bisect this to the libexpat1-udeb update as described in this mail to
debian-boot@:
  https://lists.debian.org/debian-boot/2017/08/msg00078.html
Unfortunately, the black screen issue (which was fixed in the meanwhile)
masked another issue: when X starts, no input is processed at all, be it
keyboard or mouse.
I've just finished a new investigation session, building the netboot-gtk
image against buster's udebs across many timestamps (snapshot.debian.org
for the win!), copying the old (known good) libexpat1-udeb manually to
take it out of the picture, and managed to reach this conclusion:
  20170720T035111Z: last OK
  20170720T160309Z: first KO
Comparing udeb versions between builds for both dinstalls is trivial,
since there are only two differences:
  -libudev1-udeb 233-10
  +libudev1-udeb 234-1
  -udev-udeb 233-10
  +udev-udeb 234-1
Rebuilding against buster at the 20170720T160309Z timestamp with the old
udev udebs makes d-i work properly again. Rebuilding against current sid
with the old udev udebs makes d-i work properly again.
I've only quickly glanced at the contents of both packages, and debdiff
mentions no obvious issues (file lists are the same).
I haven't looked into the changelog and actual changes (yet), but I'd be
happy to get some input (no pun intended) from systemd maintainers.
Thanks for your time.
KiBi.
